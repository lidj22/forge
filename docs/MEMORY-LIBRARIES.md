# Agent 记忆方案选型

---

## 1. 专用记忆框架对比

| 框架 | 核心能力 | 记忆策略 | 多模型支持 | 开源 | 推荐场景 |
|------|---------|---------|-----------|------|---------|
| **Mem0** | 通用记忆层，自动提取事实/偏好，向量+图谱存储 | 语义记忆、图谱记忆、自动去重冲突解决 | Claude/Gemini/OpenAI/Ollama | Apache 2.0 | **最佳全能选择**，50k+ 开发者 |
| **Zep** | 时序知识图谱（Graphiti），追踪事实随时间变化 | 三层图谱 + 渐进式摘要 + 时序失效 | Claude/OpenAI/Gemini | Graphiti 开源，Cloud 托管 | 需要追踪演变的长期分析 |
| **Cognee** | 知识引擎，原始数据→结构化知识图谱 | 双层（会话记忆+永久知识）、自优化图谱 | Claude/OpenAI/Gemini | 开源 | 重度知识管理 |
| **Letta (MemGPT)** | Agent 自主管理记忆（类似 OS 虚拟内存） | 核心记忆（常驻）+ 归档记忆（检索）+ 回忆记忆 | OpenAI/Claude | 开源 | Agent 自主决定记什么忘什么 |
| **LangMem** | LangGraph 生态的记忆 SDK | 扁平 KV + 向量搜索，需自建存储 | 继承 LangChain 全部 | 开源 | 已在 LangChain 生态中 |
| **Motorhead** | Rust 构建的轻量记忆服务器 | 滑动窗口 + Redis 向量搜索 | LLM 无关（纯存储） | 开源 | 最简单的方案，已不太活跃 |

## 2. 框架内置记忆

| 框架 | 记忆能力 | 适合 |
|------|---------|------|
| **LangChain Memory** | 最全的经典策略：Buffer、滑动窗口、摘要、实体、向量检索 | 需要多种经典记忆策略 |
| **LlamaIndex Memory** | Token 限制短期记忆 + 自动溢出到长期、事实提取、向量块 | Agent 需结合文档检索 |

## 3. 向量数据库（作为记忆后端）

| | ChromaDB | Pinecone | Weaviate |
|---|---|---|---|
| 定位 | 轻量嵌入数据库 | 全托管向量库 | 向量+图谱混合 |
| 开源 | Apache 2.0 | 否 | BSD-3 |
| 最适合 | 本地开发、小规模 | 生产大规模 | 混合搜索、多模态 |
| 成本 | 免费 | 付费 | 免费+托管 |

## 4. 与 My Workflow 的结合方案

### 推荐：Mem0 作为核心记忆层

```
┌─ My Workflow ──────────────────────────────┐
│                                            │
│  Session (记忆策略: full)                   │
│       │                                    │
│       ▼                                    │
│  ┌─ Mem0 ─────────────────────┐           │
│  │  自动提取事实和偏好          │           │
│  │  向量存储 (ChromaDB)        │           │
│  │  图谱记忆 (实体关系)         │           │
│  │  跨会话记忆共享             │           │
│  └────────────────────────────┘           │
│       │                                    │
│       ▼                                    │
│  不同策略映射到不同 Mem0 配置：              │
│                                            │
│  none           → 不调用 Mem0              │
│  sliding_window → 原始消息窗口（不用 Mem0）  │
│  full           → Mem0 提取+存储+检索       │
│  full_summary   → Mem0 + 用 Gemini 生成摘要 │
│  external       → Mem0 + 直接写入 Obsidian  │
└────────────────────────────────────────────┘
```

### 为什么选 Mem0

1. **最成熟** — 50k+ 开发者，YC 支持，有研究论文
2. **多模型** — 原生支持 Claude/Gemini/OpenAI，可以用免费模型做记忆提取
3. **开源** — Apache 2.0，可以本地部署，数据完全自控
4. **轻量** — 不需要额外的图数据库（除非开启图谱记忆）
5. **Token 节省** — 号称 80% prompt token reduction

### 备选：Zep（如果需要时序追踪）

对于市场分析类的长期会话，事实会随时间变化（"竞品 X 上季度收入 5000 万 → 这季度 8000 万"），Zep 的时序知识图谱能自动处理这种更新和失效。

可以**两者混用**：
- 大多数会话 → Mem0（通用记忆）
- 分析/调研类会话 → Zep（时序图谱）
